#Walmart Sales Data Analysis

Overview

This project focuses on analyzing Walmart’s sales data to uncover trends, patterns, and actionable insights that can help improve business decision-making.
The primary goal is to understand which branches and product lines perform best, how customer behavior varies across locations, and what factors influence overall sales performance and profitability.

The dataset used for this project comes from the Kaggle Walmart Sales Forecasting Competition
, which provides historical sales data across multiple Walmart stores and departments.

Project Objectives

Analyze branch-wise and product-wise sales performance.

Identify sales trends across time, weekdays, and months.

Understand customer purchasing patterns and their impact on revenue.

Explore the relationship between pricing, tax (VAT), and profit margins.

Methodology
1. Data Preparation

Performed data cleaning and validation to ensure consistency and completeness. All missing or null values were checked and handled appropriately.

2. Feature Engineering

Created additional attributes to enrich the analysis:

time_of_day – Categorized sales into Morning, Afternoon, and Evening.

day_name – Extracted weekday names to analyze daily sales patterns.

month_name – Identified the month of each transaction for trend analysis.

3. Exploratory Data Analysis (EDA)

Conducted descriptive and comparative analysis to identify high-performing products, profitable months, and active customer segments using SQL queries and data visualization techniques.

4. Insights & Reporting

Summarized key findings and patterns to highlight areas for strategic improvement in sales and customer engagement.

Tools & Technologies

SQL – For data extraction, transformation, and analysis

Python (Pandas, Matplotlib) – For data manipulation and visualization

Excel – For exploratory summaries and validation
